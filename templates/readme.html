Problem Definition & Objective

The Problem: Every day, millions of people open streaming platforms and spend a frustrating amount of time scrolling through hundreds of movie titles without knowing what to watch. The problem is not a lack of movies — it is the absence of a reliable, personalized guide.

Project Objective

To build a content-based movie recommendation system that accepts any movie title as input and returns a ranked list of the ten most similar movies. Similarity is determined by combining five complementary content signals extracted from rich movie metadata.

Why Content-Based Filtering?

The TMDB/IMDB dataset provides per-movie metadata rather than per-user rating logs. This makes collaborative filtering impossible, as there is no user behavior records to model. Instead, we extract and combine five expressive content signals that together describe the complete identity of a film:

  • Genre labels — the category of film (Action, Drama, Sci-Fi).
  • Plot keywords — specific thematic elements (heist, dream, dystopia).
  • Creative personnel — directors (triple-weighted), writers, and cast.
  • Overview text — full plot synopsis vectorized with bigrams.
  • Numeric metrics — vote average, count, revenue, runtime, popularity.

Relevance & Impact

Recommendation systems are among the most commercially valuable AI applications. Netflix states its engine drives over 80% of content watched, saving over $1 billion annually. This project simulates a real-world content-based pipeline at academic scale using a dataset of 367,909 movies.

Data Cleaning & Preprocessing

3.1 Filter to Released Films

Only Released films are meaningful for recommendations. Unreleased films have no audience history.

df = df[df['status'] == 'Released'].copy()

3.2 Remove Nulls & Duplicates

df.dropna(subset=['title'], inplace=True) df.drop_duplicates(subset=['title'], keep='first', inplace=True)

3.3 Filter Top 50,000 by Vote Count

The vast majority of raw movies have zero or near-zero votes. We retain the 50,000 most-voted movies to ensure meaningful similarity scores and manage memory.

df = df.nlargest(50_000, 'vote_count')

3.4 Director Triple-Weighting

The director's creative identity is the single strongest discriminating signal. The director token is inserted three times to give it higher TF-IDF weight.

def people_str(row): parts = [] if row['directors']: d = clean(row['directors']).replace(' ', '_') parts += [d, d, d] # triple weight # ... add writers and cast ... return ' '.join(parts)
04

Exploratory Data Analysis

Analysis was conducted on the filtered 50,000-movie working set to validate design decisions.

  • Vote Count: Heavily right-skewed. Log-scaling (log1p) is essential to prevent blockbusters from dominating the numeric signal.
  • Genre Distribution: Drama and Comedy are the most common (>20%). 19 distinct labels ensure TF-IDF produces discriminating vectors.
  • Keywords: Median of ~8 keywords per movie. Bigram TF-IDF captures compound concepts like "artificial intelligence".
  • Directors: Top 10 directors account for ~4% of the set, validating the decision to triple-weight director tokens.
05

Feature Engineering

The Memory Problem: A naive implementation would compute pairwise similarity for each signal, producing five N x N matrices. For N=50,000, this requires 50 GB of RAM — impossible in Colab. The solution: store normalized vectors, not scores.

The 5-Signal Architecture

We fuse five complementary signals into one weighted sparse matrix.

Genre (TF-IDF) W: 0.25
Content guardrail ensuring thematic relevance. Rare combinations get higher IDF weights.
Keywords (Bigram) W: 0.20
Plot-level specificity. Captures compound concepts like "based on comic".
People (Director x3) W: 0.20
Creative identity. Strongest discriminating signal. Matches auteur styles.
Overview (Bigram) W: 0.15
Thematic NLP similarity from plot synopsis.
Numeric Metrics W: 0.20
Quality and cultural impact signal. Log-scaled to handle skew.

Model Building

Offline Training

Training runs once in Colab and produces four serialized artifacts.

pickle.dump(combined, open('combined_matrix.pkl', 'wb'), protocol=4) pickle.dump(movie_meta, open('movie_meta.pkl', 'wb'), protocol=4) pickle.dump(title_index, open('title_index.pkl', 'wb'), protocol=4)

Query-Time Inference

Finding top-N recommendations requires a single sparse matrix-vector multiplication. Because rows are L2-normalized, the dot product equals cosine similarity exactly.

idx = title_index[movie_title] query_vec = combined[idx] scores = (combined @ query_vec.T).toarray().flatten() scores[idx] = -1 # exclude self top_idx = np.argpartition(scores, -n)[-n:]

08

Results & Insights

Key Findings

  • Director triple-weighting is the most impactful design decision for auteur filmmakers.
  • Bigram TF-IDF on keywords outperforms unigrams for semantic concepts.
  • Log-scaling numeric columns is essential to prevent popularity bias.

Limitations

  • No personalization: The system is item-to-item only, not user-specific.
  • Cold-start: New films with zero votes have a weaker numeric signal.
  • Genre granularity: Labels are coarse (e.g., Horror covers many sub-genres).
